Samsung Supply Chain BI Dashboard
This project is a data warehouse and reporting solution built using Excel and Power BI. The data was first structured into fact and dimension tables in Excel, then imported into Power BI to create interactive dashboards.

Data Model
The warehouse follows a star schema with the following tables:

Dimension Tables

Customers

Products

Suppliers

Facilities

Dates

Fact Tables

Inventory

Procurement

Production

Sales

Shipments

This structure supports analysis across customers, products, suppliers, inventory, procurement, production, and shipments.

Dashboard Overview
After importing the Excel data into Power BI, a Supply Chain and Logistics Dashboard was prepared. It provides a clear view of performance across the supply chain stages: Supplier, Inventory, Shipment, and Customer.

Key Metrics
Gross Revenue: 186.86M

Total Revenue: 176.95M

Profit: 48.56M

Profit Margin: 27.44%

Perfect Order %: 75%

Total Shipments: 8K

Supply Chain Stages
Supplier: 129K order quantity, with breakdown by supplier (Samsung, BOE Technology, Sony, Taiwan).

Inventory: 160K inventory value, with breakdown by product (Galaxy series).

Shipment: 3M shipment quantity, with carrier delays tracked (DHL, FedEx, CMA, DB Schenker, etc.).

Customer: 187K delivered quantity, with revenue by customer (Amazon, Flipkart, Best Buy, MediaMarkt, Samsung).

Visuals
Bar charts for supplier orders, inventory value, shipment delays, and customer revenue.

KPI cards for revenue, profit, margin, perfect order %, and shipments.

Icons representing each stage for clarity (factory, cargo ship, delivery van, house).

Workflow
Data prepared in Excel and structured into fact and dimension tables.

Imported into Power BI.

Relationships defined between tables.

Dashboard visuals created for KPIs and supply chain stages.

Outcome
The dashboard provides end-to-end visibility across Samsung’s supply chain, helping track supplier performance, inventory health, shipment delays, and customer revenue contribution.
